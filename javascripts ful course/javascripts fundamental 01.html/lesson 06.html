STRING AND TEMPLATE LITERALS

STRING LITERALS

string literals are sequences of characters enclosed within single quotes (' '), double quotes (" "), or backticks (` `). They are used to represent textual data.

String literals can contain letters, digits, special characters, and whitespace. They can also represent empty strings by enclosing nothing between the quotes or backticks

  • Single quotes:
    • let singleQuotes = ' This is a string with single quotes. '
  • Double quotes:
    • let doubleQuotes = " This is a string with double quotes. "
  • Backticks:
    • let backticks = ` This is a string with backticks. `

Rules and Convention of String Literals

  1. Quoting Styles:   Javascript allows the use of single, double, and backtick quotes for string literals.
  2. Escape Sequences:   certain characters can't be friendly included in a string literal and must be escaped using backlashes (). Common escape sequences include \n for newline, \t for tab, and \ for a single backslash.
  3. Concatenation:   Strings can be concatenated using the + operator or by placing them next to each other.

TEMPLATE LITERALS

Template literals are a feature of Javascript that allow for more flexible string formatting and manipulation compared to traditional string literal.

FEATURES OF TEMPLATE LITERALS.

  1. Backtick Syntax:   Template literals are enclosed within backticks (` `), rather thatn single or double quotes
  2. String Interpolation:   Template literals allow for easy interpolation of variables, expressions, and even function calls within ${} placeholders inside the string.
    • let name = 'Janet';

      let stringInterpolation = `My name is ${name}`;
  3. Multline Strings: Template literals can span multiple lines without the need for explicit line breaks or concatenation, making multine strings more.
    • let name = 'Janet';

      let multilineString = `My
      name
      is
      ${name}`;
  4. Expression Evaluation:   Expression inside ${} placeholders are evaluated, allowing for dynamic content generation within the string.
    • let a = 100;
      let b = 200


      let expressionEvaluation = ` The sum of ${a} and $b{b} is ${a + b}; `

Template literals provide a more concise and readable way to work with strings in Javascript, especiall when dealing with dynamic content or multine string.

RULES AND CONVENTIONS OF TEMPLATE LITERALS:

  1. Backticks Usage: Template literals must be enclosed within backticks (` `) rather than single or double quotes.
  2. String Interpolation: Placeholders ${} inside the template literals can contain variables, expressions, or function calls. These placeholders are replaced with their corresponding values during runtime.
  3. Multiline String: Template literals can span multiple lines without requiring explicit line breaks or concatenation. This improves code readability, especially for multiline strings.
  4. Escape Sequence: backlashes () can be used within template literals to escape backticks or to include special characters.
  5. Expression Evaluation: Expressions inside ${} placeholders are evaluated. However, only valid Javascript expression are allowed inside the placeholders.
