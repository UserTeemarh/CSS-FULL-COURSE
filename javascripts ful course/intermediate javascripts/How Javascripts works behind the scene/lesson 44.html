Scope and The Scope Chain

scope concepts

scoping
scoping is how our program's variables are organized and accessed. "where do variables live?" or "where can we access a certain variable, and where not?".
lexical scoping
scoping is controlled by placement of functions and blocks in the code.
scope
space or environment in which a certain variable is declared (variable environment in case of functions). there is global
scope of a variable: region of our code where a certain variable can be accessed

the 3 types of scope

Global scope
Variables declared outside any function are called global scope. global scope can be accessed from anywhere in a javascript program. mark varibles declared with var, let, and const are quite similar when declared outside a block
function scope
Function scope are not accessible (visible) from outside the function. they are also called local scope
block scope
The block scope of a variable means that the variable is accessible within the block. however, this only applies to let and const variables. functions are also block scoped (only in strict mode)

scope chain

Scope chain establish the scope for a given function. each function defined has its ownn nested scope, and any fucntion defined within another function defined within another function has a local scope which is linked to the outer function ____ this link is called the chain

SCOPE CHAIN (detailed)

difference between the scope chain and call stack

scope chain is all about the order in which functions are written in the code. while, call stack is a mechanism used to keep track of the function calls in a program
