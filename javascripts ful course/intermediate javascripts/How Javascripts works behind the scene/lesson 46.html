Variable Environment: Hoisting and the TDZ

what is Hoisting in Javascript?
hoisting makes types of variables accessible/usable in the code before they ae actually declared. in other word Javascript hoisting occurs during the creation phase of the execution context that moves the variable and function declarations to the top of their scope.
behind the scenes

before execution, code is scanned for variable declarations, and for each variable a new property is created in the variable environment object.

what is TDZ in Javascript?
TDZ (temporal dead zone) is the area of a block where a variable is inaccessible until the moment the computer completely initializes it with a value. A block is a pair of braces ( {...}) used to group multiple statements. Initialization occurs when you assign an initial value to a variable
where exactly is the scope of a temporal dead zone?
a block's temporal dead zone starts at the beginning of the block's local scope. it ends when the computer fully initializes your variable with a value.
what is the difference between temporal dead zone and hoisting?
temporal dead zone (TDZ) hoisting
temporal dead zone is the period between variable is initiated and hoisted hoisting is the invoking the variables or functions at the top of the scope no matter where it declared.
