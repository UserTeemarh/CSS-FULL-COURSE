THE “THIS” KEYWORD

What is “THIS” Keyword
"this" Keyword refers to the object that is currently executing or calling the function.. The “THIS” keyword will refer to the object to which it belongs to on runtime, depending upon where its call-site (where it is called). It is a way to access and manipulate the properties and methods of the object that called the function.
For example. imagine you have a function that is attached to a button on a web page. When you click the button, the function is called, and The “THIS” refers to the button object that was clicked

“THIS” Keyword can have the following values depending upon where it is accessed:

  • By Default: “THIS” refers to the global object
  • Inside a function: “THIS” refers to the global object. in strict mode, however, “THIS” Keyword will be undefined
  • Inside a Method: “THIS” refers to the owner object. (A method is a function that belongs inside an object. In other words, it's a function that's an object's property).
  • In an Event: “THIS” refers to the element on which the event was triggered.
  • Inside an Immediately Invoked Function Expression: (IIFE) “THIS” refers to the global object, In strict mode, “THIS” will be undefined, just like any other function in a global object
  • Inside a Fat-Arrow Function: When a fat arrow ( )=> is used to define a function, it doesn't create a new value for “THIS”, instead, it keeps referring to the same object it was referring to outside of the function

Contexts in which the “THIS” Keyword can be used:

What is Context?     Context generally refers to the value of the “THIS” keyword within a function. It represents the object to which the current code belongs. The context can be affected by how a function is involved.

  • “THIS” in the method of an object
    A method is used to refer to a function that is a member of an object. All methods are functions, but not all functions are methods. Now, when the “THIS” keyword is used inside a method, it refers to the owner of the method it is used in

    Example below

    greet: function( ){
    return " Hi! I am " + this.firstName +     " " + this.lastName + " I am a frontend developer";
    }

  • Using of the BINDINGs.
    What is BINDING:  Binding in Javascript means recording that identifier in a Specific Environment Record.  Each Environment Record is related to a specific Execution Context __ and that binds the identifier (variable or function name) to the this keyword for that execution context.

    Types of BINDINGs

    1. Implicit Binding
      Implicit Binding refers to the automatic binding of the “THIS” keyword to an object when a method is invoked on that object. It happens automatically based on the context in which a function is called

      Example below

      let blog = {
      name:   'Allan',
      Job:   'Teacher',
      OutPut:   function ( ) {
      console.log (`${this.name} is a ${this.Job}`);
      }
      };
      blog.OutPut( )

    2. Explicit Binding
      Explicit Binding involves manually specifying the context (the object to which “THIS” refers) for a function using methods like call, apply and bind
      Built-in Method in Explicit Binding
      • call( ): method is a method that allows you to invoke a function with a specified “THIS” value and arguments provided individually.   It's a way to explicitly set the value of this for a function and also pass arguments to the function.

        • Example below

          function calJohn (message) {
          console.log(`${message} ${this.john}`);
          }
          const person = {
          john: 'John'
          }
          console.log(calJohn.call(person,'Hello'));

      • apply( ): The apply( ) method is another way to perform explicit function binding. It is similar to call( ), but instead of passing arguments individually, arguments are passed as an array.

        • Example below

          function calJohn (message) {
          console.log(`${message} ${this.john}`);
          }
          const person = {
          john: 'John'
          }
          console.log(calJohn.apply(person, ['Hello']));

      • bind( ): The bind( ) method is used for explicit function binding as well, but unlike apply( ) and call( ), bind( ) doesn't Immediately invoke the function. Instead, It creates a new function with the specified this value and any intial argument. The original function remains unchanged.

        • Example below

          function calJohn (message) {
          console.log(`${message} ${this.name}`);
          }
          const persona = {
          name: 'Mckenzie'
          }
          // Using bind() to create a new function with 'persona' as the context
          const calJohnPersona = calJohn.bind(persona);
          // Invoking the new function___ (calJohnPersona)
          console.log(calJohnPersona('Hola'))

    3. Default Binding
      Default Binding occurs when a function is invoked as a standalone function, i.e, not as a method of an object. In the default binding, this inside the function typically refers to the global object. (e.g window in a browser enviroment). However, if you are in strict mode mode, the default binding for this is undefined instead of the global object.

      • function defaults () {
        console.log(this.title);
        }
        const faults = {
        title: 'Juan'
        }
        console.log(defaults());

      Understanding default binding is cruicial to avoid unexpected behaviour in your code, especially when dealing with the value of this in different contexts.

  • In an Event Handler:
    In the context of an Event Handler, “THIS” typically refers to the element that triggered the event.

    Examples below

    const myObject = {
    property:    "value",
    myMethod: function( ) {
    console.log(this.property)
    }
    }
    console.log(myObject.myMethod( )) Output: "value"

  • In a Constructor Function:
    When used in a Constructor Function, “THIS” refers to the instance of the object being created.

    function myClass (value) {
    this.value = value;
    }
    const myInstance = new MyClass (42);
    console.log(myInstance.value); // Output:    42

  • In a Function called without an Explicit Context:
    When a Function is called without an explicit context (not part of an object or not bound), “THIS” refers to the global object (window in browsers).

    function MyFunction( ) {
    console.log( this) // Refers to the global object (e.g window in a browser)
    }
    console.log(MyFunction( ));

Importance of The “THIS” in Javascript

  • Accessing Object properties: When you have an object with properties and methods, you can use the The “THIS” keyword to access and manipulate those properties from within the object's functions.
  • Reusability of Code: By using the “THIS” keyword, reusable functions can be created across different object. The same functions can be used to access and manipulate the properties of different object, depending on which object calls the function.
  • Contextual Execution: The “THIS” keyword allows us to execute functions in the context of the object that called them. This means that we can manipulate the object's properties and methods directly from within the function, without needing to pass in the object as an argument
  • Method Chaining: The “THIS” keyword is often used in method Chaining, where you can call multiple methods on an object in a single line of code. This can make your code more concise and easier to read
